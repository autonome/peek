# Peek page model

This document describes the design paradigm for a web page lifecycle in Peek..

Key pieces:

- User actions and decisions
- Page metadata
- Windowing model
- Chaining integration
- Extensibility

## Page loading user actions and decisions

As a user, I need to:

- Open a web page URL
- See the web page and interact with it
- Easily access website information, such as the URL
- See extended information, such as the SSL cert info
- See extended information, provided by extensions or customizations
